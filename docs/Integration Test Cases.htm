2         Test Scenarios

 

IPDRWriteTool doctype {XDR|XML} inputfile1.csv  . . . outfile outputFileName [recordcount recordcount] [roundrobin] {-ipdrversion 3.1/3.5 | -schemavalidation}

 

IPDRReadTool doctype {XDR|XML} inputFileName [outfiledir] outputFileName {-ipdrversion 3.1/3.5 | -schemavalidation}

 

2.1      Test Case 1

testVOIP.csv file containing 30 records with more than 20 fields is used as input to  IPDRWriteTool. This file contains schema location for VOIP schema

VoIP3.0-A.0ID.xsd schema file is used.

 

Recordcount and roundrobin options are off

 

Output xdr file is : TestCase1.xdr

Output xml file is : TestCase1.xml

Three .csv files are used as input to the IPDRWriteTool. testVOIP.csv with 30 records, testSM.csv with 40  records,  testWS.csv with 30 records  are used. All .csv files contain the location of their respective schemas.

                       

VoIP3.0-A.0ID.xsd, SM203.0-A.0.xsd, WS30.-A.0RD1.xsd schema files are used.

 

            Recordcount of 200 is used and roundrobin option is off.

            Output xdr file is : TestCase2.xdr

            Output xml file is : TestCase2.xml

            VoIP3.0-A.0ID.xsd, SM203.0-A.0.xsd, WS30.-A.0RD1.xsd schema files are used.

This is the case of nested schema. File testBestvendorSM.csv containing 100 records is used as input to

3         General Instructions

1.     Each .csv file contains the location for their respective schemas. Path of the .xsd file requires to be mentioned in the first line (Mandatory) of  .csv file. e.g. if the .xsd file is present in c:\, and if working directory is also c:\, then the path of schema location file in .csv file will be c:\filename and if the working directory is different then relative path or absolute path is required in the .csv file.

2.     Sometimes there is a problem of number format. e.g. the number may be present in scientific format like 8.96E+11. It is required to be converted in to decimal notation like 895744000000. Otherwise NumberFormatException will arise and data will not get processed.

3.     After reading the xdr or xml file, .csv file/files is/are generated.  If .csv files are opened for view, the same should be closed before doing read again.
