Linux Build Instruction for gcc Compilation of the IPDR File Transfer Protocol Libraries

Dependent Libraries

The reference libraries were built and tested using version 7.9.3 of libcurl for URL implementation and version 1.95.2 of Expat for the XML parser.  It is also dependent on the base IPDR library.

Build Instructions

Below are more detailed steps for compilation on the Linux platform.  Other Unix flavors should be similar.

Note: if you have built the c_ipdrbase on the same system, then steps 1 and 2 a. can be skipped.
  1. After downloading and unpacking curl and expat, you'll still need to configure make, build and install each.  For each library:
    1. execute the config script
    2. run make
    3. (possibly as root) run make install to populate the target directories with the compiled libraries.  By default on Linux this will be /usr/local/*.  Both libs and header files will be moved.
  2. To work with the default makefile for IPDRFTP you will need to:
    1. copy the expat header file from /usr/local/include/expat.h to a subdirectory, creating /usr/local/include/expat/expat.h
    2. Specify the path of ipdrlibs(conversion library) folder in file “makefile.mk” by modifying IPDRLIBSROOTPATH variable in the file.This ipdrlibs folder contain include directory, which in turn has directories named common, xdr, xml, utils. This is required by test scripts ConsumerIT.c and ProducerIT.c.
    3. Run the shell script to set some environment variables:
           . ./setcpath.sh
    4. create a subdirectory under the <ipdrftp>directory for <ipdrftp>/libs
    5. run make at the <ipdrftp> directory
  3. Modify your classpath add the ipdrftp/test directory.  This will allow you to simply execute the IPDRReadTest or IPDRWriteTest from any directory.
