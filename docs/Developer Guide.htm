2          Goal of Reference Libraries

 

The objective of the reference libraries is to provide software implementers of the IPDR standards with libraries and tools to reduce construction cost, increase quality, and increase interoperability of their IPDR facilities.

 

These libraries simplify the tasks of reading and writing IPDR data. The libraries will be

useful to both consumers and producers of IPDR records who wish to encode or decode XML or XDR documents and exchange them using either the IPDR file-based transfer protocol or their own file or socket-based transfer implementation.

The libraries are available in ANSI C and Java and currently support NDM-U version 3.5

3.1       Data Structures

 

Before using the APIs it is important to understand the key input and output parameters. The key data structures are described below

 

FNFType        FNFType is the descriptor that describes the input or output data. It contains the service type and list of attribute descriptor. Each attribute descriptor has attribute name, attributes type, derived type (corresponding to schema) and description (Mandatory / Optional).

FNFData        FNFdata associates data with its descriptor. This structure contains data array and the associated FNFType Structure

·        FNFType

o       Service Name: Identified the record type defined in service definition

o       ListAttributeDescriptor: List of Attribute Descriptors.

§         Attribute Descriptor: Defines one attribute which will appear in a record

·        Attribute Name: The name of an attribute. This name will either come from the default namespace or it will come from an alternate namespace as defined in the header.

·        Attribute Type: Describes the type of this attribute.

·        Derived Type: Derived type, populated from schema otherwise will be same as Attribute Type.

·        Description: Identifies whether attribute is mandatory/ optional.

e.g. AttributeDescriptor 1

Attribute Name:  startTime

Attribute Type:    Integer

Derived Type:     dateTime

Description:         Mandatory

 

AttributeDescriptor 2

Attribute Name:  serviceProviderID

Attribute Type:    UTF8_Array

Derived Type:     UTF8_Array

Description:         Mandatory

 

 

·        ListIPDRData

o       It is a link list, each node containing raw data value.

e.g. For above attribute Descriptors example data values are:

§         AttributeDescriptor 1

Data Value = 1001325663

§         AttributeDescriptor 2

Data Value = eBroadCasting

 

Structures representation:

 

1.      typedef struct FNFData

{            

  char* typeName_;

  ListAttributeDescriptor* pListAttributeDescriptor_;

  ListIPDRData* pListIPDRData_;

} FNFData;

 

2.      typedef struct FNFType

{

    char* serviceType_;

    ListAttributeDescriptor* pListAttributeDescriptor_;

    ListAttributeDescriptor* pListServiceAttributeDescriptor_;

} FNFType;

 

3.      typedef struct ListAttributeDescriptor

{

 AttributeDescriptor* pAttributeDescriptor_;

 struct ListAttributeDescriptor* pNext_;

} ListAttributeDescriptor;

           

4.      typedef struct AttributeDescriptor

{

 char* attributeName_;

 char* attributeType_;

 char* description_;

 char* derivedType_;

 char* complexType_;

 int   noOfElements_;

 List* pListAttrNameList_;

 List* pListAttrClassList_;

 List* pListAttrMaxOccursList_;

 

} AttributeDescriptor;

 

5.      typedef struct ListIPDRData

     {

                int IPDRDataType_;

                IPDRDataUnion IPDRDataUnion_;

               int noOfElements_;

               ListIPDRArrayData *pListIPDRArrayData_; /*Array*/

               ListIPDRStructureData *pListIPDRStructureData_; /*Structures*/

                struct ListIPDRData* pNext_;

             } ListIPDRData;

 

                        where IPDRDataType can have any of the following primitive values as given below -

 

INTEGER                 0x00000021

UNSIGNED_INTEGER        0x00000022

HYPER                   0x00000023

UNSIGNED_HYPER          0x00000024

FLOAT                   0x00000025

DOUBLE                  0x00000026

HEXADECARRAY            0x00000027

STRING                  0x00000028

BOOLEAN                 0x00000029

BYTE                    0x0000002a

UNSIGNED_BYTE           0x0000002b

SHORT                   0x0000002c

UNSIGNED_SHORT          0x0000002d

ARRAY                   0x40000000

STRUCTURE               0x80000000

 

 

IPDRHeader  Represents IPDR Header Information as per NDM-U 3.5

 

The libraries have functions and utilities to populate FNFType and FNFData. They are as follows:

 

1.      ListIPDRData is a link list. Its each node contains data corresponding to basic data types.

a.       appendListIPDRData(ListIPDRData*, Raw Data, Data type)        

·        First argument is Data List.

·        Second argument is Data Value (Raw).

·        Third argument is Data Type (one out of eight basic data types).

 

e.g. appendListIPDRData(sampleListIPDRData, 1000, IPDRDATA_LONG);

           

2.      FNFType contains the service type and list of attribute descriptor. FNFType can be created using following functions.

a.      generateDescriptor (SchemaURI, FNFType*, OptionalAttributes*, ErrorCode)

This is used to generate FNFType with all mandatory attributes and selected optional attributes.

 

·        First argument is a character pointer, specifying Schema URI.

·        Second argument is a output FNFType structure (containing service type and list of attribute descriptor)

·        Third argument is list of Optional attributes (which have to be included with all the mandatory attributes).

·        Fourth argument is an integer value, indication error code in case of error.

 

e.g. generateDescriptor (schemaURI, pFNFType, OptionalAttributes, pErrorCode);

 

b.  generateSchemaDescriptor(SchemaURI, FNFType*, ListNameSpaceInfo*, ErrorCode)

      This is used to generate FNFType with all mandatory and optional attributes.

 

·        First argument is a character pointer, specifying Schema URI.

·        Second argument is a output FNFType structure (containing service type and list of attribute descriptor)

·        Third argument is output NameSpaceInfo list (containing NameSpaceURI & NameSpaceID).

·        Fourth argument is an integer value, indication error code in case of error.

 

    e.g. generateSchemaDescriptor(schemaURI, pFNFType, pListNameSpaceInfo, pErrorCode);

 

 

3.      FNFData can be created using a API exposed function createFNFData.

a.       createFNFData( IPDRCommonParameters*, FNFType*, ListIPDRData*, FNFData*, ErrorCode)

This is used to generate FNFData using FNFType and ListIPDRData.

 

·        First argument is a IPDRCommonParamters. Common structure containing all the necessary information, need to be transported across different functions in libraries.

·        Second argument is FNFType structure (generated using above written functions).

·        Third argument is ListIPDRData list (generated using above written functions).

·        Fourth argument is an integer value, indication error code in case of error.

 

    e.g. createFNFData(pIPDRCommonParameters, pFNFType, pListIPDRData, pFNFData, pErrorCode);

           

4.      Schema validation can be provided by calling function setSchemaData

a.      setSchemaData(IPDRCommonParameters*, ListNameSpaceInfo*, ListServiceDefinitionURI*, defaultNameSpace*, ErrorCode )

This can be used for setting up schema lookup table and schema flag for schema validation.

 

·        First argument is a IPDRCommonParamters. Common structure containing all the necessary information, need to be transported across different functions in libraries.

·        Second argument is ListNameSpaceInfo, (containing NameSpaceURI & NameSpaceID).

·        Third argument is ListServiceDefinitionURI (containing list of service definition URIs).

·        Fourth argument is defaultNameSpace.

·        Fifth argument is an integer value, indication error code in case of error.

 

      e.g. setSchemaData(pIPDRCommonParameters, pListNameSpaceInfo, pListServiceDefinitionURI,  defaultNameSpace, pErrorCode );

 

5.      IPDRDocWriter provides functionality to write complete IPDRDoc in XDR and XML format.

6.      IPDRDocReader provides functionality to read IPDRDoc.
