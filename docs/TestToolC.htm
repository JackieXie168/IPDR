2         Overview

 

Command line test tools are provided to test the reference libraries. The write tool can be used to generate NDM 3.5 compliant XML and XDR documents using a simple comma separated file (csv) as input. Options are provided in the write tools to generate documents from multiple service specifications as well as generate a specified load. The read tool can be used to read NDM 3.5 compliant XML and XDR documents and write these down to a csv file.

The read and write tools internally link to the reference libraries. Hence they provide a useful way to run a quick test on the libraries and validate any changes.

 

3         Write Tool

 

The write tool uses a simple comma separated file as input to generate XML and XDR documents.

 

3.1      Input csv File Format

           

·        Each csv file can contain data corresponding to one service specification only.

·        The first line of the csv file contains the location of the schema file for which the csv file contains data. This location is a URL and support http, ftp and file. An example of the URL for each of these protocols is as follows

http://192.168.122.121/nested/VoIP.xsd

ftp://192.168.122.121/nested/BestVendorSM.xsd

file://localhost/c:/ipdrlibs/SM203.0-A.0.xsd

·        The second line of the csv file contains the nameSpaceURI prefixed with nameSpaceID as nameSpaceID$nameSpaceURI.

                                  e.g. bestvendorIPDR$http://www.bestvendor.com/ipdr/namespace

                    If the namespace is the default namespace then only the namespace should be specified.

·        The third line of the csv file contains all the attribute names in the exact order as specified in schema.

·        Subsequent lines contain records to be written in the IPDRDoc in XDR or XML format.

 

 

3.2      Command Line Options

 

IPDRWriteTool doctype {XDR|XML} inputfile1.csv  . . . outfile outputFileName [recordcount recordcount] [roundrobin] {-ipdrversion 3.1/3.5 | -schemavalidation}

 

Where:

 

-doctype          This is a mandatory argument that is used to specify the type of IPDR document to be generated. The argument for this option can have a value of of XML or XDR only.

                        e.g. doctype XDR

 

Input csv files    This is a mandatory argument specifying input csv

files containing data for the write tool. Atleast one input csv file should be specified. Multiple csv files can be specified by separating by blank spaces.

e.g. Inputfile1.csv Inputfile2.csv

 

-outfile              This is a mandatory argument that takes an argument specifying the output file. This argument can contain the full path of the output file. If the complete path is not specified the current directory is assumed.

 

-recordcount    This is an optional argument that indicates the number of records to be written to the IPDRDoc.

e.g. recordcount 1000

 

-roundrobin      This flag is specified is valid only if more than one input file is specified. When roundrobin is specified records from the multiple csv files are written to the IPDR Document in a round robin fashion. In the absence of this argument the csv files are written sequentially in the order they are listed on the command line.

 

-schemavalidation          This is a mandatory argument, if -ipdrversion is not present. This argument is used to specify whether data will be validated against the schema.

 

-ipdrversion 3.1/3.5          This is a mandatory argument, if -schemavalidation is not present. This argument is used to specify the version.

 

3.3      Sample Tool Usage Scenarios

 

1.      Both recordcount and roundrobin options are not present

In this case records of the csv files will be written in the same order as given in the usage. e.g.

     

IPDRWriteTool doctype XML testSM.csv testVOIP.csv testWS.csv outfile IPDRDoc -schemavalidation

             

In this case, records of testSM.csv will be written first followed by records 

of testVOIP.csv  followed by records of testWS.csv.

 

2.      recordcount option is present and roundrobin option is off

-recordcount is the number of records to be written in IPDRXMLDoc.xml.  Hence the tool  keeps iterating sequentially through the files till the specified number of records are written out. e.g.

 

IPDRWriteTool doctype XDR testSM.csv testVOIP.csv testWS.csv -outfile IPDRDoc -schemavalidation recordcount 320

            

If testSM.csv contains 30 records, testVOIP.csv contains 40 records and testWS.csv contains 30  the tool will go through each file 3 times and then write 20 records from testSM.csv to write a total of 320 records.

 

                                         

3.      recordcount option is off and roundrobin option is on

      

When roundrobin is specified records from the multiple csv files are written to the IPDR Document in a round robin fashion.  e.g.

 

IPDRWriteTool doctype XDR testSM.csv testVOIP.csv testWS.csv outfile IPDRDoc -schemavalidation roundrobin

 

In this case, first record of testSM.csv will be written to the IPDRDoc followed by first record of testVOIP.csv followed by first record of testWS.csv. Similarly, second record of testSM.csv followed by second record of testVOIP.csv followed by second          record of testWS.csv and so on.

 

 

4.      Both recordcount and roundrobin options are on

 

            In this case records from the csv files will be written iteratively in a roundrobin fashion until recordcount number of records are written out. e.g.

 

IPDRWriteTool doctype XDR testSM.csv testVOIP.csv testWS.csv outfile IPDRDoc -schemavalidation recordcount 320 roundrobin

              

Considering 30 number of records in testSM.csv, 40 number of records of testVOIP.csv and 30 number of records of testWS.csv. First record of testSM.csv followed by first record of testVOIP.csv followed by first record of testWS.csv and so on.

 

 

4         Read Tool

 

The read tool is primarily used to convert IPDRDoc in XDR and XML format to raw data and then writing these out to a user friendly comma separated (csv) file format. Input to the Read Tool is IPDRDoc in XDR or XML format. Only one IPDRDoc can be specified as input to the Tool. The csv file / files generated will have the same characteristics as defined under the  Write Tool.

                                    

The output csv files will be named according to the service type (complex type) present in the schema from which data is generated. There is no provision to specify the output file name on the command line.

 

4.1       Command Line Options

 

IPDRReadTool doctype {XDR|XML} inputFileName [outfiledir] outputFileName {-ipdrversion 3.1/3.5 | schemavalidation}

 

Where:

 

-doctype          This is a mandatory option that is used to specify the type of IPDR document to be generated. The argument for this option can have a value of of XML or XDR only.

                        e.g. doctype XDR

           

 

inputFileName  This is a mandatory argument that specifies the input IPDR File. Only one file can be specified which must be in XML or XDR format. Directory path for input file needs to be specified. If Directory path is NOT specified, current directory is used.

 

-outFileDir        This is an optional argument that specified the directory path in which output .csv files will be generated.

                        e.g. outFileDir c:\ipdrlibs\org\ipdr\test\

                        If absent the current directory is used.

 

-schemavalidation         This is a mandatory argument, if -ipdrversion is not present. This argument is used to specify whether data will be validated against the schema.

 

-ipdrversion 3.1/3.5          This is a mandatory argument, if -schemavalidation is not present. This argument is used to specify the version.
