State transitions for client and server

This document gives extra explanation of the IPDR protocol states.

IPDR Protocol States

The IPDR protocol defines the following states corresponding to a IPDR session.
DISCONNECTED IPDR Collector or Exporter are not connected on both the Transport Layer and Protocol layer
CONNECTED IPDR Collector and Exporter are connected on the Transport Layer
SESSION INITIATION IPDR Collector and Exporter are connected on the Protocol Layer, the templates for connection have not been determined
READY IPDR Collector has proper templates for the session and awaits for starting Active Session
ACTIVE SESSION IPDR Collector has Active Session with Exporter

The IPDR State Machines

The operation of the IPDR Client and Server are modeled as state machines, which is described by the following state transition tables. Any implementation of a IPDR Client or Server must conform to the procedures specified by the table. The table does not exhaustively show all the state transitions that may occur at a IPDR Client or server, but the major transitions are outlined.
