This document describes the integration test case scenarios for which sample input csv files,range files,capability files and the generated output files using reference libraries have been provided.

 

The procedure for conducting integration tests:

 

The integration test cases files are lying in the file IPDRFTP_testData.tar (For Unix) and IPDRFTP_testData.zip (For Windows). Unzipping or untaring these files results in the creation of a test folder. A number of sample Capability files have also been provided in the Test folder. Apart from these files, the test folder also contains some folders, these are the control file directories, for sample SubscriptionGroups as given in the CapabilityFile. The URL for this file has to be specified for running any of the test cases.

Change the value of the controlFileDirectory(for Consumer) and controlFileLocalDirectory( for Producer) nodes to point to the exact location and URL of the control file directory, for the SubscriptionGroup otherwise the test cases will generate a sequence of errors. Before this, check for the existence of control file directories. If they do not exist, create the controlFile directories at the desired location, for each of the SubscriptionGroups. Inside each of these control file directories, create a range file with a name, confirming to the CapabilityFile values. For example, for the SubscriptionGroup with groupID “Voip”, create a range file with the name “Voip-range-file” (name obtained as <groupID>-range-file) in the directory “Voip”. Make an entry as “0-“ in each of the range files. If these paths are different from the one specified in the CapabilityFile, change the capability file to point to exact controlFile directory location and URL.

Test cases for  integration of conversion libraries with the file transfer libraries.

 

1.      For producerIT:

1.      Place the csv files at a location or the csv files are lying in the test folder can be used.

2.      Open ProducerIT.c to edit the following variable values:

These variables are located at the beginning of the .c files.

Specify in “inputFiles” array the exact locations of the csv files. The csv files specified in this array will be read.

For example if the csv files are located at the path: “c:/ipdrftp/test/”, then specify locations in the array as :

 

char** inputFiles = (char**)malloc(sizeof(char*));

inputFiles[0] = " c:/ipdrftp/test//testSM1.csv";

inputFiles[1] = " c:/ipdrftp/test/testSM2.csv";

inputFiles[2] = NULL;

 

Specify docType as 1 in case XML is required to be written else specify docType as 0 in case XDR is required to be written.

 

int docType = 0;

 

Specify in outFile the exact location of the directory in which xml or xdr should be created.

 

char* outFile = "c:/ipdrftp/test/";

 

3.      Run the command “make clean” and “make all”

4.      Run the ProducerIT.c by running the command ./producerIT

5.      This creates xml or xdr files in the directory specified in the ipdrDocName variable.

 

2.      For consumerIT:

1.                  Open the ConsumerIT.c for editing the following variables, specified at the beginning of the class.

 

Specify docType as 0 in case XML is required to be read else specify docType as 1 in case XDR is required to be read

 

int docType = 0;

 

Specify in outDir the exact location of the directory in which csv files will be created.

 

char* outDir = "c:/ipdrftp/test/";

 

            Run the following commands:

            “make clean”

            “make all”

            ./consumerIT

 

2.                  Consumer should run normally and should create two csv files inside the directory specified in the outDir variable.
