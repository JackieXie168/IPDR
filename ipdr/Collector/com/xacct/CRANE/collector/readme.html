The classpath must include jdom.jar and xerces.jar. These are in the Lib directory, which also has CRANEcollector.jar.

From the top-level directory, you can do:

CLASSPATH='Lib/CRANEcollector.jar:Lib/jdom.jar:Lib/xerces.jar' java com.xacct.CRANE.collector.Test \ crane/com/xacct/CRANE/collector/CRANEcollector_skeleton.xml 30 csv (For Windows: "CLASSPATH='Lib/CRANEcollector.jar;Lib/jdom.jar;Lib/xerces.jar'").

To compile (on Windows; for Unix use ":" instead of ";"):

mkdir -p classes2 javac -g -O -target 1.3 -classpath "Lib/jdom.jar;Lib/xerces.jar" -sourcepath crane \ -d classes2 crane/com/xacct/CRANE/collector/*.java

To get configuration information from the client, after editing the xml file to have an appropriate client IP address (or name, if the client is defined by DNS):

java com.xacct.CRANE.collector.Test CRANEcollector_skeleton.xml 30 config >config.xml To run the server, using the "config.xml" file: java com.xacct.CRANE.collector.Test config.xml 30 csv (where "csv" can be replaced by "null" or "null0" or "dedup")

For information about running the simulator, see the top-level "readme".
