Build Instructions

Below are more detailed steps for compilation on the Linux platform.  Other Unix flavors should be similar.

  1. After downloading and unpacking curl and expat, you'll still need to configure make, build and install each.  For each library:
    • execute the config script
    • run make
    • (possibly as root) run make install to populate the target directories with the compiled libraries.  By default on Linux this will be /usr/local/*.  Both libs and header files will be moved.
  2. To work with the default makefile for IPDR you will need to:
    • copy the expat header file from /usr/local/include/expat.h to a subdirectory, creating /usr/local/include/expat/expat.h
    • run config in the IPDR install directory
    • create a subdirectory under the install directory for <ipdrinstalldir>/libs
    • run make in the install directory
  3. Modify your classpath add the ipdr/test directory.  This will allow you to simply execute the IPDRReadTest or IPDRWriteTest from any directory.
  4. Test the program from a shell by doing the following:
    • cd <ipdrinstalldir>/test/testdata/testcase1
    • Execute the write tool to create an IPDR XML file:
> IPDRWriteTool -doctype XML -outfile testVOIP.xml -schemavalidation testVOIP.csv
    • Execute the write tool to create an IPDR XDR file:
> IPDRWriteTool -doctype XDR -outfile c_testVOIP.xdr -schemavalidation testVOIP.csv
    • Use the read tool to validate the results:
> IPDRReadTool -doctype XML -schemavalidation testVOIP.xml

> IPDRReadTool -doctype XDR -schemavalidation testVOIP.xdr

Known Bugs:

  • If Readtool is called with just the arguments "-doctype XML" it will crash rather than report a missing parameter error.
    Workaround:  provide a complete argument string
